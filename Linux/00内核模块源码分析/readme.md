如何阅读内核源码
	=> 背景介绍
		=> 了解该模块/机制的背景，为何要引入该模块/机制，该模块/机制是用来解决什么问题的？
		=> 基础概念
	=> 相关的文件
		=> 相关的代码位于内核的什么位置
	=> 重要的数据结构
	=> 重要的函数
	=> 类图和顺序图(C++/JAVA)
	=> 调用流程
补充:
	=> 阅读代码最重要先了解其架构，然后逐步细化，从宏观到微观。注意把握阅读代码时颗粒度的大小，黑箱/白箱。
	=> 除了阅读代码之外，搜集尽可能多的文档，帮助理解
	
另外，如果是分析C++/JAVA等面向对象的语言，类图和顺序图可以很好的帮助分析。
有一些工具可以从源码直接生成类图或顺序图(逆向工程)，如: Enterprise Architect, Rational Software Architect。

看源代码有两种方式:
	=> 自上而下，看基本 use case
	=> 自下而上，看基本 function 功能
